사용자의 현재위치 마커에는 "내 위치"라는 문구를 마커위에 띄워서 현위치 가독성 증가시키기

로그인 되지 않은 사용자는 마커를 찍을 수 없게 만들기
마커를 찍으려고 하면 로그인 후 이용할 수 있는 기능입니다 나오게 하기 alert X  -> Modal 이용

리스트에서 가게를 누르면 해당 가게의 위치에 마커를 찍기 
( 사용자에게 해당 가게의 위치를 보여주기 위한 기능 , 위치만 보여주는 것 DB에는 저장 X )

마커 누르면 해당 마커를 지도의 중앙에 위치하게 움직이기

지도를 클릭하면 클릭된 해당 지점을 지도의 중앙에 위차하게 움직이기

모든 사용자에게 보이는 마커에 어떤 사용자가 마커를 남긴건지 표시
( 여러 사용자가 중복된 장소를 마커표시할 수도 있으므로 여러명의 이름이 나올 수 있게 만들기 ) 

지도에 나와 있는 상세 가게를 누르면 해당 가게 정보 나오게 ( 리스트에서 가게 누르면 나오는 정보창과 같게 )

내 현재 위치로 이동하는 버튼 만들기

지도 안쪽의 우측에 마우스로 줌인/줌아웃 뿐만이 아닌 클릭으로 줌인/줌아웃 기능 추가

마커삭제를 하면 즉시 마커가 삭제되지 않음

모든 마커의 전화번호가 정보 없음으로 나옴

리스트에서 마커를 추가하면 카테고리가 잘 나오지만, 지도에서 마커를 추가하면 카테고리가 미분류로 나오고 있음

마커추가를 하면 해당 마커의 버튼이 즉시 마커삭제로 바뀌어야하는데 그러지 않음

로그인 하기 전에는 마커에 여러 명의 사용자가 구분자( 콤마 ) 없이 붙어서 나오고 있음 + 로그인 후에는 여러 명의 사용자가 나오는 것이 아닌
로그인된 사용자가 최우선으로 나옴 ( 한 명만 나오는 것이 문제 ) 

마커 추가후 추가한 사용자 지도에 반영 안됨 ( DB에는 바로 들어가나, 마커추가직후 나오는 모달창에서는 나오되 모달창 닫았다가 다시 마커를 눌러 정보를 보면 방금 추가했던 사용자가 사라져 있음 )

마커 삭제하면 해당 마커를 일단 지도에서 지워버림 ( 마커를 찍은 다른 사람이 있던 말던 일단 지움 ) + DB에서는 다른 사람은 지워지지 않음
즉 , 지도에서만 일시적으로 삭제됨

본인이 추가한 마커에는 사용자정보가 나오지 않음 + 여러명의 사용자가 마커를 추가하면 사용자정보가 나오지 않음

리스트에서 클릭하고 닫기 누르면 마커 없애기 

리스트에서 가게 누르면 infowindow 나오는거 삭제 + panto 추가하기

마커 추가/삭제 시 페이지 리디렉션 되지 않고 버튼 바뀌고 마커 바로바로 삭제되게 만들기

검색 입력칸의 오른쪽 부분이 구분선과 딱붙어있음. 검색버튼 크기랑 같게 조정.

리스트에서 나오는 마커는 사용자가 구분할 수 있게 다른 색깔로 표현하기

검색으로 나오는 마커는 리스트에서 나오는 마커와 동일하게 초록색으로 표시.

화면 가로비율 1:3 으로 나눈다음에 1에 검색칸, 3에 지도 넣기 + 지도크기 키우기
만약 가게이름을 검색했다면 현재 위치에서 가장 가까운 가게를 보여주기
장소는 해당 장소의 중심을 보여주기
키워드로 장소검색하고 목록으로 표출하기

내위치 ~ 마커까지의 거리 표시해보기

마커를 눌렀을 때 나오는 사용자의 이름을 누르면 해당 사용자의 마이페이지로 이동하게 되어서 해당 사용자의 정보를 볼 수 있음
마이페이지에서 해당 사용자가 작성한 리뷰를 볼 수 있게 해야함. 

지도에서 보이는 모달창에 평점 추가해야함. 만약 사용자 리뷰가 없다면 리뷰없음 으로 표시

일단 검색,리스트에서 가게를 누르면 잠깐 평점이없는 모달창이 나왔다가 평점이 있는 모달창으로 변경됨.
지도하고 마커클릭으로 나오는 모달창에는 평점이 존재하지 않음

기존 마커 누르면 panto 추가하기 > 로그인된 사용자가 즉시 추가한 마커에는 panto가 적용되지만, 예전에 넣어뒀거나 다른 사용자가 작성한 마커에는 panto 적용안됨. -> mapObj 가 생성이 안되어서 null 값 전달중. 위도 경도를 기반으로 한 좌표는 잘 전달됨

사용자가 마커를 추가한 가게에 대한 리뷰를 남길 수 있게 만들기 ( 마커를 추가하지 않으면 리뷰 쓸 수 없음 )

리뷰의 점수를 기반으로 장소의 총 평점 계산하기( 해당 장소에 대한 리뷰 총점 / 해당 장소에 대한 리뷰 총 개수 )

등록한 사용자 한명만나오고있음 ( 다시 오류남 )

리뷰를 남길때 place_id 를 받아오는 이유로 같은 장소에 대한 리뷰를 두 명의 사용자가 작성해도 places테이블의 서로 다른 place_id로
받아와서 별점의 계산이 이루어 지는 것이 아니라 각기 다른 장소에 대한 별점이 되어버림

마커삭제하면 평점 다시 계산해야함.

리스트하고 검색에서 평점이있는 모달창이 한번에 빡 나오는게 아니라 평점이없는 모달창이 먼저나왔다가 바뀜
리스트하고 검색은 평점이 나오지 않았으면 좋겠음.
